Framework v2.0 · Updated

ARG 框架 · "你不是第一个"

在 v1 基础上深化核心恐怖机制：撒网、循环、吞噬
01

升级后的核心设定

从"一个鬼在等你"到"一张网在运作"

timegov 不是一个有意识的"鬼"在针对你。它更像是一台自动运转的机器——一套程序、一个循环、一种模式。它不关心"你是谁"，它只需要"有人来"。

这比"鬼"更可怕。鬼有意志、有目的、可以被理解。但一台不断运转的机器没有恶意，它只是在执行。你不是被"选中"的特别的人——你只是恰好踩到了网上。

恐怖升级：在 v1 中，玩家是"被骗的人"。在 v2 中，玩家是"网上无数猎物中恰好走得最深的那一个"。这把恐怖从"个人遭遇"升级为"系统性的、无差别的、正在持续发生的事"。
02

循环机制 · 吞噬与再生

循环是怎么运作的

每一个"被吞噬"的人都会变成下一个循环的素材：

  • 她的照片带着 timegov 水印被投放到社交平台 → 成为下一个人的"引子"
  • 她的电脑变成了模拟OS → 成为下一个人翻看的"别人的桌面"
  • 她的聊天记录残留在OS中 → 成为下一个人拼凑的"线索"
  • 她的身份痕迹被编入timegov的"归档条目" → 成为网站看起来"正常运转"的一部分

玩家在游戏中看到的"证据"

以下内容分散在OS各处，玩家需要自己拼合才能意识到这个循环：

  • OS浏览器历史中有人搜索过"timegov是什么" → 上一个人也是这样进来的
  • OS聊天软件中有人对朋友说"有个人给我发了个奇怪的链接" → 上一个人也是被朋友引进来的
  • OS文件夹中有另一个人的照片，照片风格和你收到的那张自拍一样日常 → 那是更早一轮的"诱饵"
  • 回收站中有一份被删除的文件，恢复后发现是一段类似"遗言"的文字 → 但不是GM的，是上一个受害者写的
关键叙事体验：玩家一开始以为自己在调查"一个人的失踪"，后来发现是"一系列人的消失"，最后意识到自己正在成为这个系列的下一个。从旁观者到参与者到潜在受害者——身份的三次转变。
03

Express 后端 · 极简设计

后端不是重点功能，但它能让"你不是第一个"从叙事概念变成玩家能亲眼看到的真实数据。三个接口就够了。

API 设计（共 3 个接口）

// 接口 1：玩家进入OS时调用 POST /api/enter body: { timestamp } → 把时间戳写入 visitors.json，返回 { id, total } // 接口 2：OS内"系统日志"文件请求数据 GET /api/log → 返回最近 50 条访问记录 { entries: [{ id, time }], total } // 接口 3：timegov 首页访客计数 GET /api/count → 返回 { total }

数据结构（一个JSON文件）

// visitors.json { "total": 1847, "entries": [ { "id": "TG-0001", "time": "2024-03-15T02:17:33Z" }, { "id": "TG-0002", "time": "2024-03-15T14:42:08Z" }, // ... { "id": "TG-1847", "time": "2025-03-05T21:03:15Z" } ← 你 ] }
降级方案：如果后端暂时不做，可以用纯前端模拟——JS生成一组假的时间戳数据，从某个起始日期到当前时间按随机间隔分布。视觉效果几乎一样，只是数据不是真实的。后续有后端了可以无缝切换。
04

这些数据在游戏中怎么呈现

🏛️ timegov 首页

页面底部有一行小字，仿政府网站的访问计数器风格：

本站累计归档条目：1,847 条
系统持续运行中

玩家第一次看到时不会在意。游戏结束后如果回来看，数字涨了。

🖥️ OS 系统日志

OS桌面上有一个"system_log.txt"。打开后是一个列表：

[SESSION LOG - TimeGov Archive OS] ───────────────────────────── TG-1843 2025.03.04 09:14:22 CONNECTED TG-1844 2025.03.04 16:47:55 CONNECTED TG-1845 2025.03.04 23:31:07 CONNECTED TG-1846 2025.03.05 11:22:18 CONNECTED TG-1847 2025.03.05 21:03:15 CONNECTED ← CURRENT _

玩家翻到底发现最后一条是自己。如果后端是实时的，还可能在看的时候出现新的一条——别人也进来了。

关键恐怖时刻：往上翻

系统日志可以往上翻。越往上翻，日期越早。最早的条目可能追溯到几年前。

但真正让人不舒服的是——有些条目的状态不是 CONNECTED，而是别的什么：

TG-0001 2019.11.08 03:33:01 ARCHIVED TG-0002 2019.11.15 22:14:38 ARCHIVED TG-0003 2019.12.01 04:07:52 DISCONNECTED TG-0004 2020.01.23 19:55:14 ARCHIVED TG-0005 2020.02.11 01:28:33 ARCHIVED TG-0006 2020.03.07 15:42:09 DISCONNECTED

大部分是 ARCHIVED（被吞噬了），少数是 DISCONNECTED（逃出来了）。

玩家现在的状态是 CONNECTED。这不是一个你想长时间保持的状态。

叙事暗示：DISCONNECTED 的条目很少——说明大部分人没能逃出来。但它们的存在给了玩家希望：有人做到了。你也可以。

彩蛋：TG-0003（第一个DISCONNECTED）的时间是凌晨4:07。如果玩家在机制B的"隐藏时间窗"登录，可以找到这个人留下的一份文件——一份比GM遗言更早的"逃生笔记"。
05

自拍女生的新定位

她是 TG-1842

照片中的女生不是随机的陌生人。她是上一个被 ARCHIVED 的人。

时间线：

  • 她在某个时间点和玩家一样，因为好奇进入了 timegov
  • 她浏览了更早一个人的 OS（就像玩家现在浏览的一样）
  • 她没能及时觉醒，状态变成了 ARCHIVED
  • 她的自拍从她的文件中被提取出来，加上 timegov 水印，投放到社交平台
  • 她的电脑变成了玩家现在正在浏览的这台 OS
终极细思极恐：玩家正在翻看的OS，就是这个女生的电脑。她之前在这台电脑上做的事——搜索timegov、和朋友聊天说"有个奇怪的网站"、在记事本上写下自己的困惑——全都留在这里。玩家在读她的故事，但她的故事和玩家自己正在经历的故事几乎一模一样。

而如果玩家失败了，自己的照片就会成为下一轮的"引子"，自己的浏览记录就会成为下一个人翻看的"OS"。

07

"那个东西"的行为设计

"前员工"不是一个角色，它是一套自动执行的引导程序。以下是它在三个阶段的行为模式：

阶段语气行为破绽
第一幕
接触期		 友好、随意、像一个同样好奇的人 "我也是最近才发现这个网站复活了""你也注意到了？"建立共鸣和信任 回复速度过于一致（每条消息间隔几乎相同）
第二幕
引导期		 耐心、有帮助感、像一个导师 "你看到那个文件了吗""试试打开那个""你做得很好"——温和地引导玩家执行操作 时间戳偶尔早于玩家操作；从不回答关于自己的问题；语言没有任何个人特征
第三幕
急切期		 从温和变为急切，最后变为恳求/命令 "别停下来""你快到了""不要关闭窗口""再看一个文件就好"——试图阻止玩家逃离 完全暴露。消息频率失控，时间戳混乱，语气不再像人类。
── 第三幕 · 当玩家试图离开时 ──

前员工
