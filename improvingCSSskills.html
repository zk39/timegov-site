从零构建 Windows 98 Web OS 模拟器

一份面向初学者的完整前端教程 — 纯 HTML + CSS + JavaScript，无框架依赖

前言：项目概览

你将学到什么

这个项目用一个 HTML 文件模拟了一个完整的 Windows 98 操作系统界面，包含：启动动画、登录界面、桌面环境、可拖拽/缩放的窗口、文件管理器、记事本、浏览器、右键菜单、开始菜单、关机动画。

通过这个项目，你会深入掌握以下技能：

技能领域具体内容
DOM 操作动态创建/销毁/更新元素，不依赖任何框架
事件系统mousedown/mousemove/mouseup 协作、事件冒泡与 stopPropagation
CSS 布局Flexbox 嵌套、position absolute/fixed、z-index 层级
状态管理用纯 JS 对象管理窗口状态、文件系统、导航历史
架构设计模块化代码组织、单例模式、持久化 DOM 策略

核心架构思想：创建一次，原地更新

💡 关键理念
很多初学者会在状态变化时用 innerHTML = "" 清空所有 DOM 再重新创建。这会导致：
① 拖拽句柄断开（元素被销毁了）
② 页面闪烁（销毁再重建有视觉跳帧）
③ textarea 输入丢失（元素被替换了）

正确做法：DOM 元素创建后保持存活，通过修改属性（style、textContent、display）来更新 UI。只有窗口关闭时才销毁对应的 DOM。

第一章：骨架与布局

1.1 HTML 结构

整个项目只有一个 HTML 文件。HTML 部分极简 — 只有一个空容器 <div id="app">，所有内容都由 JavaScript 动态创建：

<!-- 这就是全部 HTML -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Windows 98 Simulator</title>
  <link rel="stylesheet" href="https://unpkg.com/98.css">
  <style> /* 你的 CSS */ </style>
</head>
<body>
  <div id="app"></div>
  <script> /* 你的全部 JS */ </script>
</body>
</html>
💎 98.css
98.css 是一个开源库，提供了 Win98 风格的 UI 组件：按钮、标题栏、输入框、树形菜单、状态栏等。它让你不需要手写这些复古样式。

1.2 CSS 基础样式

先写好最基础的 CSS。关键是让 html → body → #app 形成满屏链条：

/* 满屏基础 */
* { box-sizing: border-box; margin: 0; padding: 0; }
html, body { width: 100%; height: 100%; overflow: hidden; }
/* ↓ 这一行非常关键！没有它，所有子元素的 height:100% 都会失效 */
#app { width: 100%; height: 100%; }

/* 桌面区域 — teal 背景只在这里，不覆盖任务栏 */
#desktop-area {
  flex: 1;          /* 占满除任务栏外的所有空间 */
  position: relative; /* 窗口用 absolute 定位在这里面 */
  background: #008080;
  overflow: hidden;
}
⚠️ 踩坑警告
如果忘记写 #app { height: 100% }，整个页面会塌缩成 0 高度，你会看到一片空白或者只有任务栏可见。这是因为 CSS 的 height: 100% 需要每一层父元素都有明确高度。

1.3 DOM 工具函数 h()

我们不用 React/Vue，但需要一个快捷方式来创建 DOM 元素。h() 函数就是我们的"迷你框架"：

function h(tag, attrs, ...children) {
  const el = document.createElement(tag);

  // 处理属性
  if (attrs) {
    for (const [k, v] of Object.entries(attrs)) {
      if (k === "style" && typeof v === "object")
        Object.assign(el.style, v);     // style 对象直接合并
      else if (k.startsWith("on"))
        el.addEventListener(k.slice(2).toLowerCase(), v);  // onClick → click
      else if (k === "className")
        el.className = v;
      else if (k === "innerHTML")
        el.innerHTML = v;
      else
        el.setAttribute(k, v);         // 其他当 HTML 属性处理
    }
  }

  // 处理子元素
  for (const c of children) {
    if (c == null) continue;
    if (typeof c === "string")
      el.appendChild(document.createTextNode(c));
    else if (Array.isArray(c))
      c.forEach(x => x && el.appendChild(x));
    else
      el.appendChild(c);
  }
  return el;
}

// 用法示例：
const btn = h("button", {
  className: "my-btn",
  onClick: () => alert("clicked!"),
  style: { color: "red" }
}, "Click Me");

// 等价于手写：
// const btn = document.createElement("button");
// btn.className = "my-btn";

// btn.style.color = "red";

第二章：启动动画

2.1 Boot 黑屏文字流

启动画面就是在黑色背景上逐行打印文字，模拟 DOS 启动过程。核心是用 setTimeout 递归实现逐行输出：

function showBootScreen(onDone) {
  const app = document.getElementById("app");
  app.innerHTML = "";  // Boot 阶段可以清空，因为之前没有需要保留的元素

  const scr = h("div", { className: "boot-screen" });
  app.appendChild(scr);

  const msgs = [
    "Starting Windows 98...",
    "Initializing system devices...",
    "Loading device drivers...",
    "HIMEM is testing extended memory...done.",
    // ...更多启动消息
  ];

  let idx = 0;
  function tick() {
    if (idx >= msgs.length) {
      // 所有消息打完，淡出后进入下一阶段
      setTimeout(() => {
        scr.style.opacity = "0";  // CSS transition 会平滑淡出
        setTimeout(onDone, 500);
      }, 1500);
      return;
    }

    // 追加一行文字
    scr.appendChild(h("div", null, msgs[idx]));
    scr.scrollTop = scr.scrollHeight;  // 自动滚到底部
    idx++;

    // 不同阶段用不同的速度，模拟真实启动
    const delay = idx > 20 ? 400 : idx > 15 ? 200 : idx > 10 ? 100 : 50;
    setTimeout(tick, delay);
  }

  setTimeout(tick, 500);  // 延迟半秒开始
}

第四章：桌面环境

4.1 桌面 + 任务栏分离

⚠️ 常见错误
很多人把 teal 背景色设在整个页面上，导致任务栏也被覆盖。正确做法是用 Flexbox 让桌面和任务栏成为两个独立的区块。 
// 外层容器：竖向排列
const container = h("div", {
  style: {
    width: "100%", height: "100%",
    display: "flex",
    flexDirection: "column"  // ← 关键！竖向排列
  }
});

// 桌面区域：flex:1 占满剩余空间
const desktopArea = h("div", { id: "desktop-area" });
// CSS: #desktop-area { flex: 1; background: #008080; position: relative; }

// 任务栏：固定 40px 高度
const taskbar = Taskbar.create();
// CSS: .taskbar { height: 40px; flex-shrink: 0; background: #c0c0c0; }

container.appendChild(desktopArea);
container.appendChild(taskbar);
┌──────────────────────────────┐ │ #desktop-area │ ← flex: 1, background: #008080 │ (teal 背景) │ 窗口都是 position:absolute 在这里面 │ │ │ │ │ │ ├──────────────────────────────┤ │ .taskbar │ ← height: 40px, background: #c0c0c0 │ [Start] [窗口按钮] [时钟] │ flex-shrink: 0 (不压缩) └──────────────────────────────┘

4.3 任务栏与时钟

任务栏是一个 const 单例对象。它的按钮区域在每次打开/关闭窗口时用 updateButtons() 重建，但外壳（Start按钮、时钟）只创建一次：

const Taskbar = {
  btnArea: null,  // 按钮容器

  create() {
    this.btnArea = h("div", { /* flex:1 占满中间 */ });
    // 返回组装好的任务栏元素
  },

  // 只重建按钮区域 — 不碰 Start 和时钟
  updateButtons() {
    this.btnArea.innerHTML = "";  // 只清空按钮区域
    for (const id of App.state.openList) {
      // 为每个窗口创建一个任务栏按钮
      const isActive = App.state.activeWin === id;
      this.btnArea.appendChild(h("button", {
        className: "taskbar-btn" + (isActive ? " active" : ""),
        onClick: () => /* 切换窗口 */
      }, info.title));
    }
  }
};

第五章：窗口系统（核心）

这是整个项目最关键的部分。

5.1 窗口外壳 Shell

每个窗口都由 createWindowShell() 创建，返回一组控制方法。窗口内容放进 bodyEl：

function createWindowShell(id, title, opts, pos, { onClose, onMinimize }) {
  // bodyEl 是内容容器 — 各应用往这里塞自己的 UI
  const bodyEl = h("div", {
    className: "window-body",
    style: { margin: "0", flex: "1", display: "flex", flexDirection: "column" }
  });

  // 标题栏
  const titleBar = h("div", { className: "title-bar" }, /* ... */);

  // 窗口主体 — 用 transform 定位
  const el = h("div", {
    className: "window",
    style: {
      position: "absolute",
      transform: `translate(${pos.x}px, ${pos.y}px)`,
      display: "flex", flexDirection: "column",
    }
  }, titleBar, bodyEl);

  // 返回控制接口
  return {
    el,       // DOM 元素
    bodyEl,   // 内容容器
    setTitle(t) { titleText.textContent = t; },
    setActive(isActive) { /* 切换标题栏样式 */ },
    setZ(z) { el.style.zIndex = z; },
    show() { el.style.display = "flex"; },
    hide() { el.style.display = "none"; },
    destroy() { el.remove(); },
  };
}
💡 为什么返回方法而不是直接操作？
这种模式叫 "接口隔离"。外部代码只通过 show()/hide()/setZ() 等方法操作窗口，不需要知道内部 DOM 结构。以后你想改窗口内部实现（比如换成 CSS class 切换），只需要改 shell 内部，不影响任何调用方。

5.2 拖拽系统

拖拽是 mousedown → mousemove → mouseup 三件套。关键设计：全局只有一个 mousemove 监听器，不会因为窗口多了而堆积事件：

const Interaction = {
  mode: null,       // null | "drag" | "resize"
  targetEl: null,   // 当前被操作的窗口 DOM
  posObj: null,     // 窗口的位置对象 { x, y, w, h }
  offsetX: 0,
  offsetY: 0,

  // 在标题栏上注册 mousedown
  makeDraggable(windowEl, handleEl, posObj) {
    handleEl.addEventListener("mousedown", (e) => {
      if (e.target.tagName === "BUTTON") return; // 点按钮别拖
      this.mode = "drag";
      this.targetEl = windowEl;
      this.posObj = posObj;
      this.offsetX = e.clientX - posObj.x;
      this.offsetY = e.clientY - posObj.y;
    });
  },

  // 只调一次 — 全局监听
  init() {
    document.addEventListener("mousemove", (e) => {
      if (this.mode === "drag") {
        this.posObj.x = e.clientX - this.offsetX;
        this.posObj.y = e.clientY - this.offsetY;
        this.targetEl.style.transform =
          `translate(${this.posObj.x}px, ${this.posObj.y}px)`;
      }
    });
    document.addEventListener("mouseup", () => {
      this.mode = null;
    });
  }
};
⚠️ 开窗动画不能用 transform
如果窗口出现动画用了 animation: ... transform: scale(0.95)，它会在动画期间覆盖定位用的 transform: translate(x, y)，导致窗口先闪到 (0,0) 位置。解决方案：动画只用 opacity：
@keyframes windowFadeIn { from { opacity:0 } to { opacity:1 } }

5.3 缩放与最大化

缩放原理同拖拽，但要区分 8 个方向（n/s/e/w/ne/nw/se/sw）。每个方向是一个隐形的 <div> 贴在窗口边缘：

/* CSS — 以东南角为例 */
.resize-se {
  position: absolute;
  bottom: -4px;
  right: -4px;
  width: 12px;
  height: 12px;
  cursor: se-resize;
}

最大化是保存当前位置/尺寸，然后设为桌面区域的宽高：

toggleMaximize() {
  if (pos.maximized) {
    // 恢复
    const r = pos.savedRect;
    pos.x = r.x; pos.y = r.y; pos.w = r.w; pos.h = r.h;
    // 更新 DOM style...
  } else {
    // 保存当前状态，然后撑满
    pos.savedRect = { x:pos.x, y:pos.y, w:pos.w, h:pos.h };
    pos.maximized = true;
    const da = document.getElementById("desktop-area");
    pos.w = da.clientWidth;
    pos.h = da.clientHeight;
    // 更新 DOM style...
  }
}

5.4 窗口层级管理

用一个递增的 highZ 计数器管理层级。点击窗口 → bringToFront() → highZ++，设为新值：

bringToFront(id) {
  this.state.highZ++;
  info.pos.z = this.state.highZ;
  info.shell.setZ(this.state.highZ);  // 修改 el.style.zIndex
  this.state.activeWin = id;
  this._syncActive();   // 更新所有窗口的标题栏样式（active/inactive）
  Taskbar.updateButtons();  // 更新任务栏按钮高亮
}

第六章：虚拟文件系统

6.1 数据结构设计

文件系统就是一个嵌套的 JavaScript 对象。每个条目是 { name, type:"folder"|"file", children?, content? }：

const FileSystem = {
  drives: {
    "C:": [
      { name: "Windows", type: "folder", children: [
        { name: "System", type: "folder", children: [
          { name: "kernel32.dll", type: "file", content: "[Binary]" },
        ]},
      ]},
      { name: "Documents", type: "folder", children: [
        { name: "todo.txt", type: "file", content: "Buy milk..." },
      ]},
    ],
    "D:": [ /* ... */ ],
  },
};

6.2 路径解析

resolve(drive, pathArray) 沿路径数组逐层深入找到目标文件夹：

resolve(drive, pathArray) {
  let entries = this.drives[drive];
  for (const name of pathArray) {
    const folder = entries.find(e => e.name === name && e.type === "folder");
    if (!folder) return [];
    entries = folder.children;
  }
  return entries;  // 返回该文件夹下的所有条目
}

// 例如：resolve("C:", ["Documents"]) → 返回 Documents 下的文件列表

6.3 文件读写

saveFile(drive, pathArray, fileName, content) {
  const folder = this.resolve(drive, pathArray);
  const existing = folder.find(e => e.name === fileName);
  if (existing) {
    existing.content = content;  // 覆盖
  } else {
    folder.push({ name: fileName, type: "file", content });  // 新建
  }
}

第七章：文件管理器

7.1 Explorer 布局

文件管理器窗口由固定部分（菜单栏、工具栏、地址栏、状态栏）和可变部分（侧边栏、文件区域）组成。固定部分创建一次不动，可变部分在导航时重建内容：

7.2 文件夹树（侧边栏）

用 98.css 的 .tree-view 和 HTML <details>/<summary> 实现可展开的树：

function buildTree(items, parentPath, drv) {
  const ul = h("ul");
  for (const entry of items.filter(e => e.type === "folder")) {
    const entryPath = [...parentPath, entry.name];
    const li = h("li");

    const det = h("details");
    det.appendChild(h("summary", {
      onClick: (e) => {
        e.preventDefault();         // 阻止默认展开行为
        navigateTo(drv, entryPath);  // 改为导航到该路径
      }
    }, entry.name));

    // 递归构建子树
    det.appendChild(buildTree(entry.children, entryPath, drv));
    li.appendChild(det);
    ul.appendChild(li);
  }
  return ul;
}

7.3 文件网格与导航

导航状态用一个对象管理，包含历史记录以支持前进/后退：

const nav = {
  drive: "C:",
  path: [],          // 当前路径，如 ["Documents", "Work"]
  selected: null,    // 当前选中的文件名
  history: [...],    // 导航历史（前进/后退用）
  histIdx: 0,
};

function navigateTo(drive, path) {
  nav.drive = drive;
  nav.path = [...path];
  nav.history.push({ drive, path: [...path] });
  nav.histIdx = nav.history.length - 1;
  refresh();  // 只重建 sidebar 和 filePane 的内容
}

第八章：记事本

8.1 Textarea 持久化

记事本的核心就是一个 <textarea>。因为我们的架构是"创建一次不销毁"，textarea 在窗口存活期间始终是同一个 DOM 元素，用户的输入永远不会丢失：

const textarea = h("textarea", {
  style: { width: "100%", flex: "1", resize: "none", fontFamily: "monospace" },
  onKeydown: (e) => {
    if ((e.ctrlKey || e.metaKey) && e.key === "s") {
      e.preventDefault();  // 阻止浏览器默认的保存行为
      doSave();
    }
  },
});
textarea.value = content;

第九章：浏览器

9.1 iframe 加载真实网页

每个标签页拥有一个 <iframe>。输入 URL 后自动补全 https:// 并设置 iframe.src：

function navigate(rawUrl) {
  let url = rawUrl.trim();
  if (!/^https?:\/\//.test(url)) url = "https://" + url;

  // 创建 iframe（只在第一次导航时创建，后续复用）
  if (!tab.iframe) {
    tab.iframe = h("iframe", {
      style: { width:"100%", height:"100%", border:"none" },
      sandbox: "allow-scripts allow-same-origin allow-forms allow-popups",
    });
    contentArea.appendChild(tab.iframe);
  }
  tab.iframe.src = url;
}

9.2 标签页系统

切换标签页时，只做 display:none/block，不销毁 iframe，这样已加载的页面不会重新加载：

function showActiveContent() {
  // 隐藏所有 iframe
  for (const t of tabs) {
    if (t.iframe) t.iframe.style.display = "none";
  }
  // 显示当前 tab 的 iframe
  const tab = tabs[activeTab];
  if (tab.iframe) tab.iframe.style.display = "";
}

9.3 连接失败的错误页面

很多网站通过 X-Frame-Options 头部禁止 iframe 嵌入。我们用 setTimeout 做超时检测，8 秒内没有 onload 触发就显示经典的 IE 错误页面：

    tab.errorEl.appendChild(buildErrorPage(domain, url));

  }
}, 8000);

第十章：右键菜单与开始菜单

右键菜单是单例的 — show() 先调用 hide() 清掉旧菜单，保证同一时间只有一个：

const ContextMenu = {
  el: null,
  show(x, y, items) {
    this.hide();  // ← 关键：先清掉旧的
    const menu = h("div", { /* 创建菜单 */ });
    document.body.appendChild(menu);
    this.el = menu;
  },
  hide() {
    if (this.el) { this.el.remove(); this.el = null; }
  }
};
⚠️ mousedown vs click 陷阱
全局用 document.addEventListener("mousedown", () => { ContextMenu.hide(); StartMenu.hide(); }) 来关闭菜单。但 mousedown 比 click 先触发！如果菜单/开始菜单没有阻止 mousedown 冒泡，点击菜单项时菜单会先被关掉，click 就打不中了。

解决方案：菜单容器要同时加 onMousedown: (e) => e.stopPropagation()。

第十一章：关机/重启/待机

Win98 的 Shut Down 对话框有三个单选按钮：Shut down / Restart / Stand by。用 _createDialog() 生成模态对话框：

// 关机 → 过渡画面 → 经典黑屏
_doShutDown() {

// 待机 → 纯黑屏，点击唤醒，直接回桌面
_doStandby() {

}

面试时可以讲的技术点

技术点怎么讲
持久化 DOM 策略"我没用虚拟 DOM 框架，而是自己实现了 DOM 元素的持久化管理。窗口创建后保持存活，通过修改属性而非重新渲染来更新 UI，避免了拖拽句柄断开和输入丢失的问题。"
