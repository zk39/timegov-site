第 1 章：启动动画（Boot Screen）

这一章做什么

打开 HTML 文件后，你会看到一个黑色背景，白色文字一行一行地打出来，模拟老电脑开机的 DOS 启动画面。最后文字打完，屏幕整体淡出变黑。

✅ 敲完你会看到：黑屏上逐行出现 "Starting Windows 98..."、"Loading device drivers..." 等文字，底部有一个闪烁的光标，全部打完后屏幕淡出。

新概念讲解

1. document.getElementById("app")

HTML 里我们写了 <div id="app"></div>，这就是我们唯一的容器。这行代码的意思是："去 HTML 里找到 id 叫 app 的元素，把它给我"。我们所有的界面都会塞进这个 div 里。

2. document.createElement("div")

凭空创建一个新的 <div> 元素（这时候它还没出现在页面上），然后用 appendChild() 把它塞进某个已经在页面上的元素里，它才会显示出来。

// 创建一个 div
const box = document.createElement("div");
box.textContent = "Hello";

// 塞进 app 容器 → 页面上才看得到
document.getElementById("app").appendChild(box);

3. scrollHeight 和 scrollTop

想象你有一个固定高度的盒子（比如屏幕），但内容比盒子高（打了很多行文字）：

┌──────────────┐ ← 盒子顶部 │ 第1行文字 │ │ 第2行文字 │ scrollTop = 0 时你看到的部分 │ 第3行文字 │ ├──────────────┤ ← 盒子底部（可见区域结束） │ 第4行文字 │ │ 第5行文字 │ 这些在盒子下方，看不到 │ 第6行文字 │ └──────────────┘ scrollHeight = 整个内容的总高度（6行全算） scrollTop = 滚动条当前位置（0 表示在顶部） 想自动滚到底部？令 scrollTop = scrollHeight 即可

在启动动画里，每追加一行文字后执行 scr.scrollTop = scr.scrollHeight，就是让文字始终"粘"在底部，最新的一行总是可见的。

4. setTimeout 递归

setTimeout(fn, 100) 表示"100 毫秒后执行 fn"。我们用它模拟"每隔一段时间打一行字"：

let idx = 0;
const msgs = ["第一行", "第二行", "第三行"];

function tick() {
  if (idx >= msgs.length) return; // 全打完了，停
  // 把 msgs[idx] 显示在页面上
  console.log(msgs[idx]);
  idx++;
  setTimeout(tick, 200); // 200ms 后打下一行
}
tick(); // 开始

这不是真正的递归（没有调用栈累积），而是"定时回调"，每次 setTimeout 结束后再安排下一次。

5. CSS transition 和 opacity

给元素加上 transition: opacity 0.5s，然后改变 opacity 的值，浏览器会自动做 0.5 秒的平滑过渡动画，不需要你自己写帧动画。

function showBootScreen() {
  // 拿到 app 容器
  const app = document.getElementById("app");
  app.innerHTML = "";  // 先清空

  // 创建黑色全屏容器
  const scr = document.createElement("div");
  scr.className = "boot-screen";
  app.appendChild(scr);

  let idx = 0; // 当前打到第几行

  function tick() {
    // 全部打完 → 等一会儿然后淡出
    if (idx >= bootMessages.length) {
      setTimeout(function() {
        scr.style.opacity = "0";  // 触发 CSS transition 淡出
        // 淡出动画结束后，可以做下一步（第 2 章会用到）
      }, 1500);
      return;
    }

    // 创建一行文字
    const line = document.createElement("div");
    line.textContent = bootMessages[idx];
    scr.appendChild(line);

    // 更新闪烁光标位置（删掉旧的，追加新的）
    const oldCursor = scr.querySelector(".boot-cursor");
    if (oldCursor) oldCursor.remove();
    if (idx < bootMessages.length - 1) {
      const cursor = document.createElement("span");
      cursor.className = "boot-cursor";
      scr.appendChild(cursor);
    }

    // ★ 关键：滚到底部，让最新一行可见
    scr.scrollTop = scr.scrollHeight;

    idx++;

    // 不同阶段用不同速度，模拟真实启动的节奏感
    let delay;
    if (idx > 20)      delay = 400;  // 最后几行慢
    else if (idx > 15) delay = 200;
    else if (idx > 10) delay = 100;
    else               delay = 50;   // 前面快

    setTimeout(tick, delay);
  }

  // 延迟 500ms 开始打字（模拟开机瞬间的等待）
  setTimeout(tick, 500);
}

// 启动！
showBootScreen();

第 2 章：登录界面

这一章做什么

启动动画结束后，自动切换到一个青色背景 + 居中的登录对话框，有用户名和密码输入框，点 OK 后显示加载动画，然后……目前就到这里（桌面下一章做）。

✅ 敲完你会看到：Boot 动画播完 → 淡出 → 出现 teal 背景的登录窗口 → 输入用户名点 OK → "Loading your personal settings..." 进度条 → 然后暂时停在那里。

新概念讲解

1. 回调函数（callback）

上一章的 showBootScreen() 在启动完成后就结束了。现在我们给它加一个参数 onDone，意思是"你完事了之后帮我执行这个函数"：

// 定义：完成后调用 onDone
function showBootScreen(onDone) {
  // ...启动动画...
  // 淡出结束后：
  setTimeout(onDone, 500);
}

// 使用：启动完成后显示登录界面
showBootScreen(function() {
  showLoginScreen();
});

这样两个界面就串起来了：Boot → Login。以后加桌面也是同样的套路：Login → Desktop。

2. 98.css 库

从这章开始引入 98.css。你只需要在 <head> 里加一行：

<link rel="stylesheet" href="https://unpkg.com/98.css">

然后用它提供的 class 名：window、title-bar、title-bar-text、window-body、field-row 等，就会自动得到 Win98 风格的外观。

3. Flex 居中

让一个东西水平垂直居中最简单的方法：

.parent {
  display: flex;
  align-items: center;      /* 垂直居中 */
  justify-content: center;  /* 水平居中 */
  width: 100%; height: 100%;
}

第 3 章：桌面环境与任务栏

这一章做什么

登录成功后进入桌面：上方是青色桌面区域（带几个可双击的图标），下方是灰色任务栏（有 Start 按钮和时钟）。两者颜色分开，不互相覆盖。

✅ 敲完你会看到：Boot → Login → 青色桌面（左上有图标）+ 底部灰色任务栏（Start 按钮 + 时钟）。双击图标会 alert 提示（窗口下一章做）。

新概念：Flexbox 布局

Flexbox 是让盒子排成一行或一列的布局方式。我们需要桌面和任务栏上下排列：

外层容器 (flex, column) ┌──────────────────────────────┐ │ #desktop-area │ flex: 1 → "我占满所有剩余空间" │ background: #008080 (teal) │ │ │ │ 📁 My Computer │ │ 📁 File System │ │ │ ├──────────────────────────────┤ │ .taskbar │ height: 40px → 固定高度 │ background: #c0c0c0 (gray) │ flex-shrink: 0 → "不要压缩我" │ [Start] 12:00 │ └──────────────────────────────┘
/* 关键 CSS */
/* 外层：纵向排列 */
.desktop-container {
  width: 100%; height: 100%;
  display: flex;
  flex-direction: column;  /* ← 子元素从上往下排 */
}

/* 桌面：吃掉所有剩余空间 */
#desktop-area {
  flex: 1;           /* "有多少空间我就要多少" */
  background: #008080;
  position: relative; /* 将来窗口会 absolute 定位在这里面 */
}

/* 任务栏：固定高度，不压缩 */
.taskbar {
  height: 40px;
  flex-shrink: 0;    /* 就算空间不够也不压缩我 */
  background: #c0c0c0;
}

flex: 1 到底什么意思？

在一个 flex 容器里，flex: 1 的意思是"在所有兄弟元素分完固定空间后，剩下的全给我"。所以任务栏占 40px，桌面区域自动占满剩下的全部高度。

flex-shrink: 0 到底什么意思？

当空间不够时，Flexbox 默认会等比缩小所有子元素。flex-shrink: 0 说"别缩我"。这样任务栏永远是 40px，不会被压扁。

第 4 章：h() 工具函数 — 告别 innerHTML

为什么需要它

到现在为止，我们创建 DOM 元素要写很多行代码：

用 h() 只需要一行：

它本质上就是对 createElement 的封装。下面的代码每一行都有注释，请仔细看：

h() 函数完整实现

function h(tag, attrs) {
  // 第 1 步：创建元素
  var el = document.createElement(tag);

  // 第 2 步：处理属性
  if (attrs) {
    var keys = Object.keys(attrs); // 拿到所有属性名
    for (var i = 0; i < keys.length; i++) {
      var k = keys[i];
      var v = attrs[k];

      if (k === "style" && typeof v === "object") {
        // style 是对象 → 逐个设置 style 属性
        // 例如 { color: "red", fontSize: "14px" }
        Object.assign(el.style, v);
      }
      else if (k.startsWith("on")) {
        // onClick → 转成 "click" 事件
        // onMousedown → 转成 "mousedown" 事件
        var eventName = k.slice(2).toLowerCase();
        el.addEventListener(eventName, v);
      }
      else if (k === "className") {
        el.className = v;
      }
      else if (k === "innerHTML") {
        el.innerHTML = v;
      }
      else {
        // 其他当 HTML 属性处理，比如 type="text"、id="xxx"
        el.setAttribute(k, v);
      }
    }
  }

  // 第 3 步：处理子元素（第 3 个参数开始的所有参数）
  for (var j = 2; j < arguments.length; j++) {
    var child = arguments[j];
    if (child == null) continue;              // null/undefined 跳过
    if (typeof child === "string") {
      el.appendChild(document.createTextNode(child)); // 字符串 → 文本节点
    } else if (Array.isArray(child)) {
      child.forEach(function(c) { if (c) el.appendChild(c); }); // 数组 → 逐个追加
    } else {
      el.appendChild(child);                  // DOM 元素 → 直接追加
    }
  }

  return el;
}
💡 arguments 是什么？
每个函数里都有一个隐藏变量 arguments，包含传给这个函数的所有参数。h("div", {style:...}, "Hello", childEl) 中，arguments[0] 是 "div"，arguments[1] 是 attrs 对象，arguments[2] 是 "Hello"，arguments[3] 是 childEl。我们从 index 2 开始遍历，拿到所有子元素。

从第 5 章开始，所有代码都会用 h() 来创建元素，你会看到界面构建变得非常紧凑。

第 5 章：可拖拽窗口

这一章做什么

双击桌面图标 → 弹出一个 Win98 风格的窗口 → 可以拖拽标题栏移动窗口 → 点 X 关闭。多个窗口可以叠加，点击窗口会把它提到最前面。

✅ 敲完你会看到：双击图标弹出窗口，标题栏拖拽可以自由移动，点最小化隐藏，点关闭销毁，任务栏出现对应按钮。

新概念：鼠标三件套

拖拽的原理是三个鼠标事件的配合：

mousedown (在标题栏上按下) │ ▼ 记录起始位置 mousemove (全局, 鼠标移动) │ ▼ 计算新位置，更新 transform mouseup (全局, 鼠标松开) │ ▼ 停止拖拽

为什么 mousemove 和 mouseup 绑在 document 上而不是窗口上？

因为鼠标移动很快时会跑出窗口范围。如果事件绑在窗口上，鼠标出界就收不到事件了。绑在 document 上，无论鼠标在哪都能收到。

transform: translate(x, y)

我们用 CSS transform 来定位窗口而不是 left/top，因为 transform 不会触发页面重排（reflow），性能更好。

拖拽公式

// 按下鼠标时，记录"鼠标位置 - 窗口位置"的差值
offsetX = mouseX - windowX
offsetY = mouseY - windowY

// 移动时，用"当前鼠标位置 - 差值"得到窗口新位置
newWindowX = mouseX - offsetX
newWindowY = mouseY - offsetY

这样窗口就会"跟着"鼠标移动，而且保持鼠标在标题栏上的相对位置不变。

// ★ 拖拽管理器 — 全局只有一组 mousemove/mouseup

var Drag = {
  active: false,
  el: null,      // 正在拖的窗口 DOM
  pos: null,     // 窗口位置对象 { x, y }
  offX: 0,       // 鼠标和窗口左上角的 X 差值
  offY: 0,

  // 让一个窗口的标题栏可拖拽
  setup: function(windowEl, titleBarEl, posObj) {
    titleBarEl.style.cursor = "move";
    titleBarEl.addEventListener("mousedown", function(e) {
      if (e.target.tagName === "BUTTON") return; // 别拖到按钮
      e.preventDefault();
      Drag.active = true;
      Drag.el = windowEl;
      Drag.pos = posObj;
      Drag.offX = e.clientX - posObj.x;
      Drag.offY = e.clientY - posObj.y;
    });
  },

  // 全局初始化（只调一次）
  init: function() {
    document.addEventListener("mousemove", function(e) {
      if (!Drag.active) return;
      Drag.pos.x = e.clientX - Drag.offX;
      Drag.pos.y = e.clientY - Drag.offY;
      Drag.el.style.transform = "translate(" + Drag.pos.x + "px," + Drag.pos.y + "px)";
    });
    document.addEventListener("mouseup", function() {
      Drag.active = false;
      Drag.el = null;
    });
  }
};
